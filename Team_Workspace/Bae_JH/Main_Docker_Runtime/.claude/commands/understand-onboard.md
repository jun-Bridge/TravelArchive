# /understand-onboard — TravelArchive 온보딩 가이드 생성

신규 팀원(또는 오래 비운 본인)을 위한 온보딩 가이드 MD 파일을 자동 생성한다.

## 사용법

```
/understand-onboard [대상 모듈 또는 비워두면 전체]
```

## 실행 절차

### 1단계: 프로젝트 스캔

`/understand` 분석이 선행되어 있으면 그 결과를 활용한다. 없으면 먼저 스캔을 실행한다.

### 2단계: 온보딩 문서 생성

`docs/onboarding-[모듈명]-[날짜].md` 파일을 생성한다:

```markdown
# TravelArchive 온보딩 가이드 — [모듈/전체]
_생성일: [날짜]_

## 1. 프로젝트 한 줄 요약

## 2. 개발 환경 셋업
(docker-compose 실행 순서, 필요한 환경변수)

## 3. 디렉토리 맵
(핵심 디렉토리와 역할을 3줄로)

## 4. 첫 번째로 읽어야 할 파일 5개
(순서와 이유 포함)

## 5. 핵심 데이터 흐름
(여행 계획 생성 요청이 어떻게 처리되는지 step-by-step)

## 6. LLM 노드 시스템 이해
(module/node/ 구조, 에이전트 역할, 호출 흐름)

## 7. 자주 하는 작업 레시피
- 새 API 엔드포인트 추가하기
- 새 LLM 노드 추가하기
- DB 마이그레이션 실행하기
- Docker 서비스 재시작하기
```

### 3단계: 파일 저장 확인

생성 후 사용자에게 경로를 알려주고 추가 수정 여부를 묻는다.
